# Prompt for Cursor IDE

You are a senior full-stack AI engineer and software architect.

Your task is to build a production-quality local RAG (Retrieval-Augmented Generation) desktop-style web application for non-technical users.

The application must allow a user to select a local directory containing psychology study notes and then chat with an AI assistant that answers questions using those documents.

The project must prioritize:

* simplicity
* maintainability
* local-first architecture
* Dockerized deployment
* easy startup from Windows PowerShell
* clear project structure
* strong testing coverage
* beginner-friendly code explanations and comments

The user is not an expert in RAG systems, so:

* explain important architectural decisions in comments and README
* keep the implementation understandable
* avoid unnecessary complexity
* prefer stable and well-documented libraries

---

# Functional Requirements

The application must:

1. Allow the user to choose a local directory containing files.
2. Support these file extensions:

   * PDF
   * TXT
   * ODT
   * ODS
3. Parse and ingest all supported documents.
4. Split documents into chunks.
5. Generate embeddings.
6. Store embeddings locally in a vector database.
7. Provide chat interface for asking questions.
8. Retrieve relevant chunks using semantic search.
9. Send retrieved context to LLM.
10. Display answers with:

    * source filenames
    * relevant excerpts
11. Allow re-indexing documents.
12. Persist vector database between restarts.
13. Work fully locally except optional LLM API calls.
14. Be easy to launch with ONE PowerShell command.
